# Online Newspaper System

## Основні можливості

* **Рольова модель:** Адміністратори, Редактори, Читачі.
* **Система підписок:**
    * Різні стратегії ціноутворення (Студентська, Корпоративна, Преміум).
    * Обмеження доступу до ексклюзивного контенту (Paywall).
* **Сповіщення:** Автоматична розсилка емейлів підписникам при виході нових статей.
* **Коментування та лайки:** Інтерактивна взаємодія з контентом.
* **Адмін-панель:** Зручне керування новинами, категоріями та користувачами.

## Технологічний стек

* **Backend:** Python 3.11, Django 5.x
* **Database:** PostgreSQL 15
* **Containerization:** Docker, Docker Compose
* **Frontend:** HTML5, CSS3, Bootstrap (Jinja2 templates)

## Використані патерни проєктування (Design Patterns)

У проєкті реалізовано ключові патерни GOF для забезпечення гнучкості архітектури:

1.  **Observer (Спостерігач):**
    * *Де використано:* Система сповіщень (`signals.py`).
    * *Як працює:* При публікації новини система автоматично сповіщає всіх підписників автора та розсилає "Breaking News".
2.  **Strategy (Стратегія):**
    * *Де використано:* Розрахунок вартості та прав підписки (`strategies.py`).
    * *Як працює:* Дозволяє динамічно змінювати логіку тарифних планів (Student vs Premium) без зміни коду моделей.
3.  **Singleton (Одинак):**
    * *Де використано:* Глобальні налаштування системи (`SystemConfig`).
    * *Як працює:* Забезпечує єдину точку керування режимом "Maintenance Mode" (Технічні роботи) для всього сайту.

## Запуск Docker (Рекомендовано)

Проєкт налаштований на автоматичне розгортання. База даних, залежності та **тестові дані** (користувачі, новини) завантажуються автоматично однією командою.

**Вимоги:** Встановлений Docker Desktop.

### Інструкція:

1.  Перейдіть у папку проєкту:
    ```bash
    cd django_project
    ```

2.  Запустіть проєкт:
    ```bash
    docker-compose up --build
    ```

3.  Зачекайте 1-2 хвилини.
    * Якщо все правильно появиться повідомлення: `web-1  | Watching for file changes with StatReloader`.

4.  Відкрийте сайт: **[http://127.0.0.1:8000/](http://127.0.0.1:8000/)**

### Як зупинити або очистити
* Зупинити сервер: `Ctrl + C`
* **Повне очищення** (якщо треба скинути базу до початкового стану):
    ```bash
    docker-compose down -v
    ```

## Локальний запуск (без Docker)

Якщо ви не використовуєте Docker, виконайте наступні кроки:

1.  Створіть віртуальне оточення:
    ```bash
    python -m venv venv
    source venv/bin/activate  # Для Windows: venv\Scripts\activate
    ```
2.  Встановіть залежності:
    ```bash
    pip install -r django_project/requirements.txt
    ```
3.  Перейдіть в папку та налаштуйте базу (SQLite за замовчуванням):
    ```bash
    cd django_project
    python manage.py migrate
    ```
4.  Завантажте тестові дані:
    ```bash
    python manage.py loaddata initial_data.json
    ```
5.  Запустіть сервер:
    ```bash
    python manage.py runserver
    ```
